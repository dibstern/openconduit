I've analyzed the authentication flow and found several issues. Here's my detailed breakdown:

The token refresh logic has a fundamental race condition. When multiple browser tabs are open simultaneously, each tab independently detects the approaching token expiry and fires a refresh request. Because the refresh endpoint doesn't implement any form of mutual exclusion or request deduplication, all tabs receive new tokens — but only the last response "wins" in localStorage, invalidating the tokens stored by earlier responses.

Additionally, the getStoredToken() helper uses Date.now() for expiry comparison rather than the server's timestamp. This means that even modest clock skew between the client and server (common in production environments) can cause premature or delayed refresh attempts.
