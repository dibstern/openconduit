Can you refactor the relay connection handler to use exponential backoff?

Here's an exponential backoff implementation with jitter for the reconnection handler. The reconnectDelay caps at 30 seconds to avoid excessive wait times:

async function reconnect(attempt: number): Promise<void> {
  const baseDelay = Math.min(1000 * 2 ** attempt, 30000);
  const jitter = Math.random() * baseDelay * 0.1;
  await new Promise(r => setTimeout(r, baseDelay + jitter));
  try {
    await this.connect();
    this.attempts = 0;
  } catch {
    this.reconnect(attempt + 1);
  }
}
