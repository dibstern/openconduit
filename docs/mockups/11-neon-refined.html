Can you refactor the relay connection handler to use exponential backoff with jitter?

I'll update the connection handler with exponential backoff and jitter. Here's the implementation:

The reconnectDelay calculation uses a capped exponential with randomized jitter:

typescript 
async function reconnect(attempt: number): Promise<void> {
  const baseDelay = Math.min(1000 * 2 ** attempt, 30000);
  const jitter = Math.random() * baseDelay * 0.1;
  const delay = baseDelay + jitter;

  // Wait before reconnecting
  await new Promise(resolve => setTimeout(resolve, delay));

  try {
    await this.connect();
    this.attempts = 0;
  } catch (err) {
    this.reconnect(attempt + 1);
  }
}

Key changes: exponential base with a 10% jitter range, capped at 30 seconds. The attempt counter resets on success.
