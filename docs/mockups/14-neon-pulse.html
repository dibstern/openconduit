The relay stack is built per-project. Each project gets its own RelayStack instance that owns an SSE consumer, session manager, and PTY bridge. Events flow from OpenCode’s SSE endpoint into relay-side caches, then fan out to connected WebSocket clients. The upstream is always authoritative — caches exist for responsiveness and recovery.
