Can you refactor the WebSocket handler to support reconnection with backoff?
ASSISTANT

I'll restructure the wsHandler to support automatic reconnection with exponential backoff and jitter. This prevents thundering-herd problems when a server recovers.

async function reconnect(attempt: number): Promise<void> { const delay = Math.min(1000 * 2 ** attempt, 30000); const jitter = Math.random() * delay * 0.1; // Exponential backoff with jitter await new Promise(r => setTimeout(r, delay + jitter)); try { await this.connect(); } catch { this.reconnect(attempt + 1); } }

The backoff caps at 30 seconds and adds 10% jitter to distribute reconnection attempts across clients.
