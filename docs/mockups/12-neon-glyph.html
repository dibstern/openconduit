Can you refactor the relay stack to use a connection pool instead of individual connections?

I can see the relay stack creates a new connection per request in relay-stack.ts:142. Here’s a connection pool implementation:

typescript 
class ConnectionPool {
  private pool: Map<string, Connection[]> = new Map();
  private maxSize: number;

  constructor(maxSize = 10) {
    this.maxSize = maxSize;
  }

  async acquire(key: string): Promise<Connection> {
    const connections = this.pool.get(key) ?? [];
    const idle = connections.find(c => !c.busy);

    if (idle) {
      idle.busy = true;
      return idle;
    }

    // Create new if under limit
    if (connections.length < this.maxSize) {
      const conn = await createConnection(key);
      connections.push(conn);
      this.pool.set(key, connections);
      return conn;
    }

    // Wait for one to free up
    return this.waitForIdle(key);
  }
}

This replaces the per-request pattern with pooled connections, reusing idle ones and capping at maxSize.
