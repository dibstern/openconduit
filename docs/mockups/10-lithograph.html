Relay architecture for
distributed AI sessions

Technical Reference — Conduit v1.0

Conduit maintains one long-lived daemon process that multiplexes browser clients across isolated project relays. Each relay owns its SSE consumer, session manager, and PTY bridge — sharing nothing with its siblings except the HTTP edge.

The event pipeline flows from OpenCode through SSE into relay-side caches, then fans out to connected WebSocket clients. State is authoritative upstream; the relay caches for responsiveness and recovery.

const relay = new RelayStack(project, client);
await relay.initialize();
daemon.mount(`/p/${project.slug}`, relay);
