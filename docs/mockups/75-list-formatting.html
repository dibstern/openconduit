Assistant

Here are the key findings:

  • Token expiry is set to 15 minutes — the default configuration uses a short-lived token with automatic refresh, but the refresh window is too narrow
  • Refresh logic has a race condition when multiple tabs are open simultaneously
  • The localStorage key is being read correctly, but the expiry check uses Date.now() instead of server time

Steps to fix:

  1. Increase the token TTL from 900s to 3600s in the auth.config.ts file
  2. Add a mutex lock around the refresh endpoint to prevent concurrent refresh calls
  3. Switch to server-issued timestamps for expiry validation to avoid clock skew
